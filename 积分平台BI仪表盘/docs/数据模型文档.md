# 积分兑换平台 - 数据模型文档

> 版本：1.0  
> 编制：BI 实训项目组  
> 适用：商务智能（BI）实训 —— 积分兑换平台

## 1. 项目背景

积分兑换平台是一个面向多商家、跨品牌的消费者积分通兑平台。消费者购买任一联盟商家的指定商品即可获得积分，积分可在平台兑换礼品。BI 系统负责：

- 商家、会员、礼品、订单 4 大主题的统计分析
- 商家会员交叉分析
- 多维度（商家 / 商品 / 会员 / 礼品 / 积分方式 / 地域 / 时间）切片

## 3. 数据仓库模型（DW 层）

### 3.1 设计原则
- 遵循 Kimball 维度建模：1 事实表 + N 维表（星型模型）
- 命名规范：`Fact_`（事实）、`Dim_`（维度）
- 代理键：所有维表用 int 自增 `*Key`；日期维额外生成 `DateKey=yyyyMMdd`
- 字段命名：源字段保留 PascalCase；维表外键加 `*Key` 后缀

### 3.3 维表（8 张）

#### Dim_Date（日期维）
参照 `BI数据仓库及ETL设计.docx` 中的 Dim_Date，字段：
- DateKey (int, PK, yyyyMMdd)
- FDate (date)
- FTheYear, FQuarter_of_Year, FMonth_of_Year, FWeek_of_Year, FDay_of_Month
- FTheQuarter, FTheMoth, FTheDay (中/繁/英)
- FHalf_of_Year, FHalfYear_cht
- FFiscal_Period, FFiveDay_of_Month
- IsWeekend, IsMonthEnd

#### Dim_Hour（小时维）
- HourKey (tinyint, PK, 0-23)
- HourName (nvarchar(10), 如 '08:00-09:00')
- TimeBucket (nvarchar(20), 早高峰/午高峰/晚高峰/其他)

#### Dim_Member（会员维）
- MemberKey (int IDENTITY, PK)
- CustomerID (int, 源系统键)
- LoginName, RealName, Gender, Phone
- RegType (int, 0:普通, 1:商家发展)
- FromBusiness (int → BusinessMen)
- RegDateKey (int → Dim_Date)
- IsValid (bit)
- IsNewMember (bit, 30 天内)

#### Dim_Merchant（商家维）
- MerchantKey (int IDENTITY, PK)
- BusinessID (int, 源键)
- BusinessCnName, BusinessEnName
- BusinessStatus (tinyint)
- CreateDateKey
- IsValid (bit)
- IsForbidden (bit)

#### Dim_Product（商品维）
- ProductKey (int IDENTITY, PK)
- ProductID, ProductName, ProductBrand, ProductType
- MerchantKey (int → Dim_Merchant)
- ProductCoin
- ProductStatus

#### Dim_Gift（礼品维）
- GiftKey (int IDENTITY, PK)
- GiftID, GiftName, GiftCategory
- GfitCoin, GiftNum
- GiftStatus
- StockLevel (int, 当前库存)
- IsHotGift (bit, 兑换次数 > 100)

#### Dim_PointType（积分方式维）
- PointTypeKey (int IDENTITY, PK)
- TypeName (nvarchar(20)): 购买商品 / 平台赠送 / 活动奖励
- TypeCode (int)

#### Dim_Region（地区维，3 级：省/市/区）
- RegionKey (int IDENTITY, PK)
- ProvinceID, ProvinceName
- CityID, CityName
- AreaID, AreaName
- RegionPath (nvarchar(200), 形如 '广东省/深圳市/南山区')

## 5. 抽取策略

- **维表**：T+1 全量拉取（数据量小）
- **事实表**：T+1 增量抽取（按 `CreateTime` 或 `InsertTime`）
- **代理键**：源键 + `ISNULL(MAX(MemberKey),0) + ROW_NUMBER()` 生成
- **缓慢变化维**：Type 2，用 `IsValid` + `EffectiveDate` + `ExpiryDate`
